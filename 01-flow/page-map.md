## 0. IA 关键决策（先读这里）

| # | 决策 | 理由 |
|---|------|------|
| IA-1 | 采用**扁平 3 层**信息架构（L0 首页 / L1 功能页 / L2 浮层或详情） | 儿童用户寻路时间 <30s，符合"3 次点击到达任何功能"硬约束 |
| IA-2 | **无全局底部 Tab** —— 唯一回路通过首页 / 顶部返回 | 答题是沉浸式体验，Tab 会打断节奏；孩子不需要导航复杂度 |
| IA-3 | **朝代图谱 = 首页 = Home**，没有独立"我的""设置"页 | 简化心智模型，所有持久态（进度/知识卡/徽章）都挂在首页可见 |
| IA-4 | 语言与主题切换放在**首页顶部次级区**（非全局悬浮），答题页仅暴露极简返回+进度 | 沉浸保护：答题过程不打扰；家长看到开关在显眼位置能放心 |
| IA-5 | 知识卡详情在**移动端 = 全屏浮层**，桌面端 = **右侧抽屉 / 居中 Modal** | 移动端空间紧张，全屏更稳；桌面端保留图谱上下文 |
| IA-6 | 错题回看是**卡片式列表 + 原题重看（不重答）** | 9-12 岁不羞辱原则，复盘而非惩罚 |
| IA-7 | 关卡解锁**只看"是否通关"**，不看分数；满分仅影响徽章 | 降低焦虑，分数不做硬门槛 |
| IA-8 | **URL 设计**：`/` `/quiz/:dynastyId` `/result/:dynastyId` `/cards` `/card/:cardId` `/wrong/:dynastyId?` | 支持深链与浏览器返回，即使无登录也要可分享/可刷新恢复 |

### 导航设计检查（PRODUCT.md 硬约束）
- [x] 用户能在 **3 次点击**内到达任何核心功能（首页 → 关卡 → 答题；首页 → 知识卡 → 详情）
- [x] 当前位置在首页图谱上用关卡状态视觉区分（锁/进行中/通关/满分）
- [x] 不需要面包屑：层级仅 2 层且返回路径唯一
- [x] 移动端不丢失桌面端入口：语言/主题/错题/知识卡四个入口两端都有
- [x] 标签用儿童语言："再来看一眼"、"朝代图谱"、"我的卡册"，而非"错题本"/"个人中心"

### IA 硬数字自检（迭代模式）

| 约束 | 阈值 | 本 IA 实测 | 状态 |
|------|------|-----------|------|
| 导航层级深度 | ≤ 3 层 | 最深 2 层（首页 → 答题 → 结算属同一任务流，非导航层级） | 通过 |
| 关键转化路径步数 | ≤ 3 步 | 首次访问 → 进第 1 关 = 2 步（首页落地 → 点击"开始"） | 通过 |
| 搜索使用率 | < 15% | 无搜索功能（MVP 内容量 = 10 关，不需要搜索） | 不适用 |
| 用户寻路时间 | < 30s | 首页一屏展示所有关卡 + 已解锁卡片入口 | 通过 |

## 2. 页面清单（Page Inventory）

### L0 · 首页 / 朝代图谱 `/`

| 字段 | 内容 |
|------|------|
| **用途** | 产品入口。展示全部 10 关状态、总进度、快捷入口；承担所有非答题场景的导航责任 |
| **关键元素** | ① 顶栏（Logo + 语言切换 zh/en + 主题切换 light/dark + 我的卡册图标）<br>② 欢迎区（首次访问时文案不同，见 State Machine）<br>③ 朝代图谱主体（10 关卡片，按时间线排列）<br>④ 总进度条（X/10 关通关，Y/10 张满分）<br>⑤ 错题回看入口（仅在 `wrongQuestions.length > 0` 时显示）|
| **入口** | 浏览器直访 `/`、任意页面点 Logo、结算页"回到朝代图谱"、首次访问 localStorage 为空 |
| **出口** | 点某关卡片 → `/quiz/:dynastyId`（仅已解锁关可点）<br>点卡册图标 → `/cards`<br>点错题入口 → `/wrong`<br>点卡片缩略 → `/card/:cardId`（L2 浮层） |
| **移动端 (375-768)** | 纵向卡片流，每卡 1 列全宽；顶栏 56px；关卡卡片 ≥48px 触摸目标；语言/主题切换在顶栏右侧 icon |
| **桌面端 (1024+)** | 10 关网格（如 5×2 或 2×5 沿时间轴），支持 hover 预览朝代简介；顶栏同构但卡册入口可显示文字 |
| **状态依赖** | 读取 `localStorage.progress` 决定每关状态；读取 `unlockedCards` 决定缩略展示 |

### L1 · 答题页 `/quiz/:dynastyId`

| 字段 | 内容 |
|------|------|
| **用途** | 核心交互：5 题一关的答题循环 |
| **关键元素** | ① 顶部返回箭头 + 关卡名 + 进度点（●●○○○ 第 2/5 题）<br>② 朝代背景/插画（移动端为轻量背景或顶部小图，桌面端为左侧主视觉）<br>③ 题干<br>④ 4 个选项按钮（纵向排列，≥48px 高，间距 ≥8px）<br>⑤ 选中后反馈层（正确/错误视觉区分 + 解释卡） + "下一题" CTA |
| **入口** | 首页点已解锁关卡、结算页点"下一关"、深链直访（需校验该关是否已解锁，否则重定向首页） |
| **出口** | 点下一题 → 同页第 N+1 题；第 5 题完成 → `/result/:dynastyId`；点返回 → 确认弹层 → 首页（本关进度清零并可重开） |
| **移动端** | 单列纵向：顶栏 → 朝代小图/背景 → 题干 → 4 选项 → 底部固定 CTA |
| **桌面端** | 两栏：左侧 40% 朝代插画 + 关卡名；右侧 60% 题干 + 选项（两栏不得丢失任何移动端信息） |
| **状态依赖** | 读/写 `localStorage.currentSession`（containing dynastyId, currentQuestionIdx, answers[]）；中途退出可恢复 |

### L1 · 关卡结算页 `/result/:dynastyId`

| 字段 | 内容 |
|------|------|
| **用途** | 通关反馈 + 奖励发放 + 下一步引导 |
| **关键元素** | ① 通关横幅（本关得分 X/5）<br>② 解锁的知识卡（新卡揭示动画，点击进 L2 详情）<br>③ 满分徽章（仅 5/5 显示）<br>④ 本关错题回看入口（仅 score<5 显示）<br>⑤ 主 CTA：下一关（已解锁第 N+1 关时）/ 回到朝代图谱（已是最后一关） |
| **入口** | 答题页第 5 题提交后自动进入；深链直访（需校验该关已完成，否则重定向首页） |
| **出口** | 下一关 → `/quiz/:nextDynastyId`；回到图谱 → `/`；本关错题回看 → `/wrong/:dynastyId`；点新卡片 → `/card/:cardId` |
| **移动端** | 纵向堆叠：横幅 → 卡片揭示 → 徽章（如有） → 错题入口 → 下一关 CTA |
| **桌面端** | 同构但卡片揭示区可居中更大，徽章在右上角浮现 |
| **状态依赖** | 写入 `localStorage.progress[dynastyId] = { completed: true, score, bestScore }`、追加 `unlockedCards`、追加 `wrongQuestions` |

### L1 · 我的卡册 `/cards`

| 字段 | 内容 |
|------|------|
| **用途** | 收集欲载体：展示 10 张朝代知识卡的解锁状态 |
| **关键元素** | ① 顶部返回 + 标题"我的卡册"<br>② 总进度（X/10 已解锁）<br>③ 卡片网格（已解锁卡显示封面 + 朝代名；未解锁显示剪影 + "通关 XX 朝代解锁"提示） |
| **入口** | 首页顶栏卡册图标、结算页点已获得卡片的缩略（直接进 L2 详情而不是列表） |
| **出口** | 点已解锁卡 → `/card/:cardId`（L2）；点未解锁卡 → 不可点或轻提示；返回 → `/` |
| **移动端** | 2 列网格 |
| **桌面端** | 3-5 列网格 |

### L1 · 错题回看 `/wrong[/:dynastyId]`

| 字段 | 内容 |
|------|------|
| **用途** | 复盘工具：查看所有答错过的题目 + 正确答案 + 解释 |
| **关键元素** | ① 顶部返回 + 标题"再来看一眼"（不用"错题本"）<br>② 筛选器（全部 / 按朝代） —— 桌面端为 Tab，移动端为下拉<br>③ 题目列表（题干预览 + 朝代标签 + 我的答案 vs 正确答案） |
| **入口** | 首页（条件显示）、结算页"本关错题回看"（带 dynastyId 参数） |
| **出口** | 点某题 → 展开或进入题目详情面板（inline expand 优先，避免再加一层路由） |
| **移动端** | 列表展开 inline，一次展开一题 |
| **桌面端** | 列表展开 inline 或左列表 + 右详情双栏 |
| **核心原则** | **只看不做** —— 不提供"重答"按钮；9-12 岁不羞辱，避免惩罚循环 |

### L2 · 知识卡详情 `/card/:cardId`

| 字段 | 内容 |
|------|------|
| **用途** | 承载单张知识卡的完整故事（人物/器物/事件 + 背景 + 一段生动文案） |
| **关键元素** | ① 关闭按钮（×）<br>② 大图/主视觉<br>③ 卡片名 + 朝代 + 分类（人物/器物/事件）<br>④ 背景故事（一段 80-150 字儿童向文案）<br>⑤ 相关朝代链接（跳回该朝代卡片状态） |
| **入口** | 首页已解锁缩略、卡册点已解锁卡、结算页新卡揭示点击 |
| **出口** | 关闭 → 回到来源页（preserve scroll） |
| **移动端** | **全屏浮层**，从底部上滑，关闭按钮左上角 |
| **桌面端** | **居中 Modal**（约 720 宽）或**右侧抽屉**（约 480 宽），保留背景图谱上下文 |
| **交互规则** | ESC 关闭、点遮罩关闭、返回键关闭（移动端需劫持 history 避免误退出整站） |

## 3. 导航模型

### 3.1 全局入口（顶栏，所有 L0/L1 保留，L2 浮层不保留）

| 入口 | 位置 | 移动端 | 桌面端 | 备注 |
|------|------|--------|--------|------|
| Logo / 回首页 | 顶栏左 | icon | icon + 文字 | 点击永远回 `/` |
| 语言切换 zh/en | 顶栏右 | icon（文字 "中/EN"） | 下拉 | 切换后保持当前页；写入 `localStorage.lang` |
| 主题切换 明/暗 | 顶栏右 | icon（☀/☾） | icon | 写入 `localStorage.theme`；跟随系统是默认项 |
| 我的卡册 | 顶栏右 | icon | icon + 文字 | 右上最显眼位置（收集欲入口） |

**答题页顶栏例外**：仅保留"返回箭头 + 关卡名 + 进度点"，**移除语言/主题/卡册切换**，避免中途切语言导致题目状态错乱（切语言需回首页做）。

### 3.2 上下文导航

| 场景 | 导航元素 |
|------|---------|
| 首页 → 朝代图谱卡片 | 每张卡片自带朝代名 + 状态徽章（锁/进行中 X/5/通关/满分） |
| 答题页内 | 进度点（●●○○○）表达"第几题 / 共 5 题" |
| 结算页 | CTA 主按钮 = 下一关（不是返回），降低跳出率 |
| 卡册 | 分朝代分组或时间轴排列 |

### 3.3 URL 与深链

| URL | 恢复策略 |
|-----|---------|
| `/` | 无论如何都可直达；localStorage 空 = 首次访问欢迎态 |
| `/quiz/:dynastyId` | 校验该关是否已解锁：未解锁 → 重定向 `/`；有未完成 session → 恢复至该 session 的题号 |
| `/result/:dynastyId` | 校验该关是否已通关：未通关 → 重定向 `/` |
| `/cards` | 无条件可达 |
| `/card/:cardId` | 校验该卡是否已解锁：未解锁 → 回 `/cards` 并提示 |
| `/wrong` `/wrong/:dynastyId` | 无错题时显示空态引导 |

## 4. 内容模型（Content Model）

| 实体 | 属性 | 关系 |
|------|------|------|
| Dynasty（朝代） | id, name_zh, name_en, order(1-10), era, illustration, shortIntro | 1:N Question; 1:1 Card |
| Question（题目） | id, dynastyId, stem_zh/en, options[4], correctIndex, explanation_zh/en | N:1 Dynasty |
| Card（知识卡） | id, dynastyId, category(person/object/event), title_zh/en, cover, story_zh/en | N:1 Dynasty |
| Badge（徽章） | id, dynastyId, type(perfect), icon | N:1 Dynasty（仅满分才获） |
| Progress（进度，客户端态） | dynastyId, completed, bestScore, attempts[] | localStorage |
| WrongQuestion | questionId, dynastyId, myAnswer, correctAnswer, timestamp | localStorage |
| Session（当前未完成关） | dynastyId, currentQuestionIdx, answers[] | localStorage，最多 1 个 |

## 6. 响应式差异总览（影响 IA 的部分）

| 页面 | 移动端 (375-768) | 桌面端 (1024+) | 差异是否影响信息可达性 |
|------|-----------------|----------------|----------------------|
| 首页图谱 | 纵向卡片流 | 网格/时间轴 | 否（10 关都可见） |
| 答题页 | 单列 | 两栏（插画+题目） | 否（信息同构） |
| 结算页 | 纵向堆叠 | 纵向堆叠 + 装饰区 | 否 |
| 卡册 | 2 列 | 3-5 列 | 否 |
| 错题回看 | 列表 inline 展开 | 列表 inline 或双栏 | 否 |
| 卡片详情 | 全屏浮层 | Modal/抽屉 | 否（细节等价） |

**两端一致性锁定**（PRODUCT.md 传递的硬约束）：
- 朝代顺序、状态视觉语言、进度表达方式两端一致
- 所有功能入口两端都必须提供（不得移动端砍功能）
- 触摸目标 ≥48px，选项间距 ≥8px（移动端硬指标，桌面端可放大）
