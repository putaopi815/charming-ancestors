# 迷人的老祖宗 · 交互说明（Interaction Brief）

> 下游读者：visual-designer（动效 token 与视觉状态）→ r-designer（Schema）→ r-pencil（.pen 演示）→ r-frontend（实现）→ r-qa（验收）

## 0. 全局交互原则（Global Principles）

**IX-P1 移动端优先、触摸优先**
- 所有交互首定义在 375×812 视口下；桌面端是"加法"（hover 预览、键盘快捷键、双栏）。
- 触摸目标 ≥ 48×48px；选项间距 ≥ 8px（SC 硬约束，任何产出物违反即阻塞）。
- 不依赖 hover 传达关键信息（移动端没有 hover）；hover 仅用于增强（tooltip、朝代简介预览）。

**IX-P2 动效三分类，禁止粗暴庆祝**（来自 PRODUCT.md 硬约束）

| 类别 | 场景 | 时长预算 | 方向 | 禁止 |
|------|------|---------|------|------|
| 发现感 | 翻开知识卡、朝代从 Locked → Unlocked、错题展开 | 400-600ms | ease-out，"推开门"意味 | 弹跳、闪烁、粒子爆炸 |
| 翻页感 | 题目 N → N+1、页面过渡、Tab 切换 | 200-300ms | ease-in-out，横向位移 + 透明度 | 3D 翻转、骤停 |
| 得到感 | 答对、解锁卡片、满分徽章 | 500-800ms | ease-out，由小到稳 | 喇叭音、"恭喜"大字、满屏烟花 |

- 所有动效 token（duration + easing）MUST 由 visual-designer 在 Design Token 中定义；本文件只给方向与预算。
- 全局尊重 `prefers-reduced-motion: reduce`：过渡降级为即时切换（duration=0）+ 轻微透明度淡入（仍保留状态反馈可读性）。

**IX-P3 儿童友好文案（来自 page-map §5）**
- 禁止用词：错、错误、失败、不正确、未通过、权限不足。
- 采用口径：
  - 答对 → "没错！"（Got It!）
  - 答错 → "再想想"（Not Quite），详情页写"正确答案是 X"而非"你错了"
  - 错题入口 → "再来看一眼"（Take Another Look）
  - 锁定 → "还没解锁"（Not Yet Unlocked）
- 所有文案参数化，禁止 `"共" + count + "条"` 拼接（i18n 硬约束）。

**IX-P4 无障碍底线（WCAG 2.1 AA + 儿童场景）**
- 所有交互状态都有 ARIA 声明（`aria-label` / `aria-pressed` / `aria-live`）。
- 焦点态可见且区别于 hover 态（焦点 = 描边 ring，hover = 底色变化）。
- 键盘可达：Tab 正序、Shift+Tab 反序、Enter/Space 确认、ESC 关闭浮层、方向键在题目/卡册网格中可用。
- 颜色不是唯一状态信息源：对/错要配 icon + 文案，锁定要配锁 icon 而不只灰调。
- 屏幕阅读器播报统一使用"朝代名 + 状态 + 可操作提示"（如"夏朝，已通关 5 题 3 分，可重玩"）。

**IX-P5 状态可恢复、无死路**
- 任何浮层均可 ESC / 点遮罩 / 物理返回键 关闭，关闭后回到来源页并保持滚动位置。
- 答题中途的返回都走 `ExitConfirm` 三选项弹层，禁止直接回首页丢进度。
- 深链非法时重定向 + toast 解释原因，不留空白页。

## 1. 首页 / 朝代图谱（`/`）

### 1.1 整体导航

**触发条件**：用户访问 `/`、点 Logo、结算页"回朝代图谱"、MidQuizPrompt 选"放弃并回首页"。
**行为描述**：
1. Boot → 读 localStorage → 分叉 FirstVisit / Returning。
2. Returning 且有 `currentSession`：**先渲染首页 DOM**，再在首屏中心弹出 MidQuizPrompt（不阻塞 DOM 绘制，避免白屏）。
3. 顶栏永远显示 Logo / 语言切换 / 主题切换 / 卡册入口 四组件。

**视觉反馈**：首次访问时 HomeFresh 态，欢迎区显示"第一次来？从夏朝开始推开历史这扇门。"，第 1 关卡片带呼吸光（duration 1600ms，loop，reduce-motion 时静态高光边）。
**动效**：入场走翻页感（200-300ms 淡入 + 轻微上移 8px）。
**错误状态**：localStorage 解析失败 → 静默降级为 FirstVisit，右下角 toast"历史记录不可用，进度不会保存"，3s 自动消失。
**无障碍**：`<main aria-label="朝代图谱">`；欢迎区 `role="status"`；首关呼吸光在 reduce-motion 下用 `outline: 2px solid semantic.accent` 替代。

### 1.2 关卡卡片四态（Dynasty Card）

| 状态 | 触摸/点击反馈 | 视觉反馈（方向） | 动效 | 可交互 |
|------|------------|----------------|------|--------|
| Locked | 轻压下沉 2px + toast"通关上一关就解锁啦" | 灰调 + 锁 icon + 朝代名斜体弱化 | 翻页感 150ms | 可点但不跳转 |
| Unlocked | 按下即进 `/quiz/:dyn` | 正常品牌色 + "开始" 文案，首关带呼吸光 | 翻页感 200-300ms 切页 | 是 |
| InProgress | 按下即恢复至 currentSession | 高亮描边（semantic.accent）+ "继续 X/5" + 进度点 | 发现感 400ms（强调这是进行中） | 是 |
| Cleared | 按下弹二次确认"要重新闯一次吗？" | 完成标记（icon，非仅颜色）+ "X/5" | 翻页感 200ms | 是（重玩） |
| Perfect | 按下弹二次确认 | 徽章 + "5/5" + 星标 | 得到感 500ms 触发徽章轻微浮动一次 | 是 |

**重玩二次确认**：
- 移动端 bottom sheet，桌面端居中 modal。
- 主 CTA"重新闯关"（危险操作色），次 CTA"取消"，默认焦点在"取消"上（防误触清进度）。
- 文案："重玩会清掉这一关的本次作答记录，满分徽章和卡片会保留。继续吗？"

**触摸目标**：卡片整体 ≥ 48×48，移动端占屏宽单列约 343×88（含内边距），桌面端 240×160 网格。
**键盘导航**（桌面端）：
- Tab 进入图谱区后走"关卡 1 → 10 → 错题入口 → 卡册缩略"顺序。
- 方向键在网格内自由移动（5×2 布局时上下键跨行、左右键跨列）。
- Enter/Space 等同点击。
**ARIA**：
- `role="link"` + `aria-label="夏朝 已通关 3/5 点击重玩"`。
- Locked 态 `aria-disabled="true"` + `aria-describedby` 指向锁定说明文本。

### 1.3 顶栏控件

**Logo**：点击回 `/`（当前就在 `/` 时无操作，但仍需 focus ring）。
**语言切换（中/EN）**：
- 移动端：icon 按钮，按下展开简单 bottom sheet（"中文 / English / 跟随系统"三选），选后立即切换。
- 桌面端：hover 展开 dropdown。
- **过渡动效**：切换瞬间正文 100ms 淡出 → 200ms 淡入新语言（翻页感变体），标题层做轻微上下位移 4px 避免"文字原地跳字数"的撕裂感。
- 切换不改 URL、保持滚动位置。
- ARIA：`aria-label="切换语言，当前 中文"`，切换后 `role="status"` 播报"已切换为英文"。

**主题切换**：
- 单按钮循环 系统 → 明 → 暗（三态轮换），或长按展开三选菜单（桌面端默认展开）。
- **过渡动效**：主题切换全屏 300ms 淡色覆盖（不是硬切也不是翻转），背景色 + 文字色同时插值，避免一段时间黑底白字的刺眼阶段。
- 图标：☀ / ☾ / 自动（字母 A 或"跟随"标记）。
- ARIA：`aria-label="主题：明亮，点击切换"`。

**卡册入口**：右上角图标 + 小红点（有新解锁未查看时）。点击进 `/cards`，翻页感过渡。

### 1.4 错题入口（条件）

**触发条件**：`wrongQuestions.length > 0` 时才渲染。
**位置**：首页主体下方次级区，一行卡片式入口，文案"再来看一眼（X 题）"。
**视觉**：禁用羞辱感配色；不用红色。用 semantic.neutral + 书签 icon。
**点击行为**：进 `/wrong`（翻页感 200-300ms）。
**空态处理**：无错题时此入口不渲染，不做"暂无错题"提示（空入口反而让孩子意识到"错题=负担"）。

### 1.5 MidQuizPrompt（未完成关卡提示）

**触发条件**：Boot → Returning → ResumeCheck 发现 `currentSession != null`，且用户是直访 `/`。
**形态**：
- 移动端：底部半屏 bottom sheet（占屏 40%）。
- 桌面端：居中 modal（宽 480）。

**三选项按钮层级（顶向下）**：
1. **继续答题**（主 CTA，品牌色填充按钮）→ 跳 `/quiz/:dyn` 恢复。
2. **先回首页看看**（次级，轮廓按钮，保留 session）→ 关闭弹层，用户下次直访 `/` 会再次提示。
3. **放弃这关**（幽灵按钮，弱化）→ 清 currentSession，关闭弹层。

**默认焦点**：打开时焦点在 1 继续答题；Enter 默认"继续"。
**ESC 行为**：关闭弹层 = 等同点"先回首页看看"（保留 session，下次再问）。
**动效**：发现感入场 400ms（由下而上滑 + 淡入），关闭用翻页感 200ms 淡出。
**ARIA**：`role="dialog" aria-modal="true" aria-labelledby="midquiz-title"`；弹层打开时 body 设 `aria-hidden` 背景。
**文案**：标题"你之前还有一关没答完呀"，副文"夏朝 进度 3/5，要继续吗？"

## 2. 答题页（`/quiz/:dynastyId`）

### 2.1 顶栏（简化，答题专属）

**组成**：返回箭头 + 关卡名（如"秦朝"）+ 进度点 `●●○○○`（第 2/5 题）。
**移除项**：语言 / 主题 / 卡册入口（见 page-map §3.1 答题页例外）。
**返回箭头触发**：进入 `ExitConfirm` 弹层。
**进度点**：纯视觉指示器，不可点（避免孩子乱跳题）；ARIA `role="progressbar" aria-valuenow=2 aria-valuemax=5 aria-valuetext="第 2 题 共 5 题"`。

### 2.2 题目展现

**触发条件**：QLoading → QReady → QShowingQuestion。
**布局**：
- 移动端单列：顶栏 → 朝代小图（高度 160，带轻渐变遮罩确保文字可读）→ 题干（字号 20-22，行高 1.5）→ 4 选项按钮。
- 桌面端两栏：左 40% 朝代插画 + 关卡名 + 第 N 题；右 60% 题干 + 选项（选项保持纵向 4 行，禁 2×2 网格——横向选项会让孩子先扫四个再决定，垂直更利阅读理解）。

**选项按钮**：
- 尺寸：移动端全宽，高度 ≥ 56px（大于 48 硬约束，儿童手指更舒适），间距 12px。
- 内部：左侧编号圆 `A/B/C/D` + 选项文本左对齐，便于口语化说"选 B"。
- 默认态：白底描边（明）/ 深底浅描边（暗），阴影极弱。
- 悬停态（桌面）：底色加深 4%，描边同品牌色。
- 焦点态：2px 外 ring（明暗双主题对比度 ≥ 3:1），键盘可见。
- 按下态：下沉 1px + 轻微缩放 0.98。

**选中后锁定**：
- 点击瞬间进入 QAnswerLocked：所有选项 `pointer-events: none`，防止重选。
- 选中项边框变粗 + 加入 icon（对：勾 ✓；错：问号 ?——不用叉，不用"X"）。
- 正确项（无论用户选了什么）**同时高亮**，让孩子看到"正确答案是这个"。

**动效**：题目入场走翻页感（N→N+1 时 horizontal slide 左移 24px 同时淡入，260ms ease-in-out）。首题入场用翻页感淡入+微上移。

**键盘**：
- 桌面端 Tab 顺序：返回箭头 → 选项 A → B → C → D → 下一题 CTA。
- 数字键 1-4 / 字母键 A-D 快捷选中（进阶，仅桌面端；移动端不适用）。
- Enter/Space 激活选项。

**ARIA**：
- 题干 `<h1>` + `aria-live="polite"` 区域。
- 选项 `role="radio"` 置于 `role="radiogroup" aria-labelledby="question-stem"`。
- 选中后 `aria-checked="true"`，正确答案同步 `aria-describedby` 指向解释文字。

### 2.3 答题反馈（对/错差异化）

> 这是全站最关键的交互。PRODUCT 明确要求"发现感而非粗暴庆祝"。

**答对（QFeedbackCorrect）**：
- 触发：用户选中的选项 === correctIndex。
- 视觉：选中项描边与底色渐变为 semantic.success（青瓷色系，不是亮绿），勾 icon 从中心 scale 0.6 → 1.0（得到感 500ms ease-out）。
- 文案：按钮下方出现一行"没错！" + 小段解释（80-120 字，儿童向）。
- 动效：得到感 500ms，**不放烟花、不震动**；仅 icon 自身的稳定浮现 + 整张答对选项轻微上浮 2px 再回位（300ms）。
- 声音：无（PRODUCT 决定）。

**答错（QFeedbackWrong）**：
- 触发：用户选中的选项 !== correctIndex。
- 视觉：
  - 用户选中项：描边加粗 + 底色轻微加深为 semantic.attention（**不是红色**，用赭石/陶土偏暖的警示色，避免"错=红=羞辱"的刻板联想），icon 为"?"。
  - 正确项：描边变为 semantic.success + 勾 icon 淡入（发现感 400ms，让孩子"发现原来是这个"）。
- 文案：选中项下方"再想想"，正确项下方"这个才对～"，并展开解释卡（80-120 字）。
- 动效：
  - 无抖动、无震动（抖动 = 惩罚感）。
  - 仅"正确项自身从暗到亮"的揭示动效（发现感 400-600ms ease-out）。
- 持久化：即刻追加到 `wrongQuestions`（保证刷新也记录）。
- **关键约束**：答错态在暗色主题下 semantic.attention 对比度 ≥ 4.5:1；勾/问号 icon 在色盲模式下仍可区分（形状不同，不仅颜色不同）。

**下一题 CTA**：
- 反馈层出现后 0.4s 显示"下一题"按钮（底部固定，移动端 sticky），避免孩子没看解释就急着点。
- 最后一题（第 5 题）文案变为"看看结果"。
- 快捷键：Enter 推进到下一题。

**ARIA（反馈态）**：
- 反馈出现时 `aria-live="polite"` 区域播报："答对了，XX 是 YY。"或"正确答案是 XX，XX 是 YY。"
- 注意不播报"你错了"——只说正确答案和解释。

**无障碍**：
- 对/错差异不仅用颜色：勾 icon vs 问号 icon 形状明确区分。
- reduce-motion 下：动效改为 100ms 硬淡入，但状态切换仍清晰可感。

### 2.4 中途退出确认（ExitConfirm）

**触发条件**：答题页点返回箭头 / 浏览器返回键。
**形态**：
- 移动端 bottom sheet（占 48%）。
- 桌面端 modal（480 宽）。

**三选项按钮层级（顶向下）**：
1. **继续答题**（主 CTA，品牌色填充）—— 关闭弹层、回到当前题。
2. **先回首页**（次级，轮廓）—— 保留 currentSession，回到首页；下次访问 `/` 会弹 MidQuizPrompt。
3. **放弃这关**（幽灵按钮，最弱化）—— 清 currentSession，回首页，答题进度从 0 开始。

**默认焦点**：**打开时焦点在"继续答题"**（防误触清进度；让键盘/屏幕阅读器用户最安全选择在首位）。
**ESC 行为**：等同点"继续答题"（最保守选项）。
**动效**：发现感入场 400ms，退出翻页感 200ms。
**文案**：
- 标题："现在离开的话..."
- 副文："这一关的进度会保留，随时可以回来继续。"（让孩子知道没关系）
**ARIA**：`role="alertdialog" aria-labelledby`；焦点陷阱（Focus Trap）。

### 2.5 异常态

**QLoadError**（题库加载失败）：
- 文案："题目加载不出来，试试看？"
- 双按钮："再试一次"（主 CTA）/ "先回首页"（次级）。
- 不说"网络错误"（儿童用户不需要技术细节）。
- ARIA `role="alert"`。

**刷新恢复**：恢复到当前题号，但当前题的选项状态清空（让孩子重新选）；顶部 toast："从第 3 题继续"，2s 自动消失。

## 3. 关卡结算页（`/result/:dynastyId`）

### 3.1 分阶段揭示（Sequential Reveal）

> 这是情感化设计核心，用"得到感"承载 9-12 岁孩子的成就体验。分阶段而不是一次性铺满，避免"信息过载"。

**阶段 1 · 得分揭示**（RRevealing → RShowingScore）
- 触发：进入结算页即开始。
- 视觉：中央大字"X/5"从 scale 0.5 淡入到 1.0（得到感 500-700ms ease-out）。
- 文案：
  - 5/5 → "全对！"
  - 3-4/5 → "不错！"
  - 1-2/5 → "好好看了卡片就更清楚啦～"
  - 0/5 → "没关系，看看卡片再来一次～"
- 禁止文案："差强人意""继续努力""不合格"等。

**阶段 2 · 知识卡解锁**（RCardReveal）
- 触发：阶段 1 动效完成后延迟 400ms 自动开始。
- 视觉：卡片背面（封缄态）从底部上浮 → 停在屏幕中央 → 翻面揭示正面（发现感 600ms ease-out）。
- **这是"发现感"的最强表达**，"推开门"的具象化：卡片翻面前有微妙的高光扫过（100ms），像光穿过门缝。
- 文案：标题"解锁了一张卡片"，卡片下方小字"点一下看看它的故事"。
- 点击已揭示卡片 → 进 L2 知识卡详情（`/card/:cardId`）。
- **重玩情形**（卡已在 unlockedCards 中）：跳过 reveal，仅显示"本关卡片你已经有啦"。

**阶段 3 · 满分徽章**（仅 5/5，RBadgeReveal）
- 触发：阶段 2 完成后延迟 300ms。
- 视觉：徽章从卡片右上角轨迹浮现并固定（得到感 500ms）。**不发光粒子、不旋转**，仅稳定浮现 + 一次轻微弹停（scale 1.1 → 1.0）。
- 文案："满分徽章 ·（朝代名）"。
- ARIA：`role="status"` 播报"获得 XX 朝满分徽章"。

**阶段 4 · 下一步引导**（RNextActions）
- 触发：阶段 2/3 完成后。
- 组成：
  - 主 CTA："进入下一关"（如非最后一关）/ "回朝代图谱"（如最后一关）。
  - 次级入口："再来看一眼（X 题）"（仅 score<5 显示）、"回朝代图谱"、"看看这张卡"（如揭示了新卡）。
- 动效：翻页感 300ms 淡入。

### 3.2 跳过动效（Skip）

- 任何阶段点屏幕任意空白处（非按钮）→ 跳到阶段 4 的静态态。
- 目的：重玩的孩子不需要再看一遍仪式；新孩子才享受仪式。
- ARIA：提供"跳过动效"隐藏链接（键盘可访问，Tab 第一项）。

### 3.3 动效总时长预算

- 完整播放：700 + 400 + 600 + 300 + 500 = **约 2.5s**。
- reduce-motion：全部降级为即时显示，直接到阶段 4 静态态；卡片揭示仍保留淡入 100ms（状态可感知）。

### 3.4 无障碍与恢复

- 浏览器后退 → 回首页（不回答题页，避免重答逻辑混乱）。
- 深链 `/result/:dyn` 但该关未通关 → 重定向 `/`，toast"这一关还没完成哦"。
- 整个流程 `aria-live="polite"` 按阶段播报（得分 → 解锁卡片 → 徽章 → 下一步）。

## 4. 我的卡册（`/cards`）与知识卡详情（`/card/:cardId`）

### 4.1 卡册列表

**触发条件**：首页卡册入口点击、深链。
**布局**：
- 移动端 2 列网格，卡片 16:9 比例。
- 桌面端 3-5 列（按视口宽度 1024/1440/1920 梯度）。

**卡片态**：
| 态 | 视觉 | 交互 |
|----|------|------|
| CardLocked | 剪影 + 灰调 + 锁 icon + "通关 XX 朝解锁" | 点击微晃 1 次（4px 左右位移，200ms）+ toast"通关 XX 朝就解锁啦" |
| CardUnlocked | 完整封面 + 朝代 + 分类 tag | 点击进 L2 详情，发现感 500ms |

**进度条**：顶部"X/10 已解锁"，展示收集感。满 10 时文案变为"全部都在你手里啦"。
**动效**：卡片入场分批淡入（每张延迟 40ms，整体翻页感 200-300ms）。
**键盘**：方向键在网格内自由移动；Enter 进详情。
**ARIA**：`role="grid"`，每张卡 `role="gridcell"` + `aria-label="夏朝 · 器物 · 已解锁，点击查看"` 或 "...未解锁，通关夏朝可解锁"。

### 4.2 知识卡详情浮层

**形态**：
- 移动端：**全屏浮层**，从底部上滑进入（发现感 500ms ease-out），关闭按钮左上角。
- 桌面端：**居中 Modal**（720×响应式高度）或**右侧抽屉**（480 宽，视觉保留朝代图谱上下文）。

**组成**：关闭按钮（×）→ 大图/主视觉 → 标题（卡名 + 朝代 + 分类）→ 80-150 字儿童向故事 → 相关朝代链接（"看看夏朝其他题"，跳回首页对应朝代卡片）。

**入场动效**：发现感 500ms（"推开门"）；背景遮罩同时 300ms 淡入到 60% 黑（明）/ 70% 黑（暗）。
**关闭路径**：
- 右上角 × 按钮（主关闭入口）。
- 点遮罩关闭。
- ESC 关闭。
- 移动端物理返回键关闭（劫持 history，避免返回整站）。
- 关闭动效：翻页感 200ms 下滑 + 淡出。

**从结算页新卡揭示点击进入**：跳过入场动效（卡片已在视觉中心，直接展开详情），避免二次仪式让孩子失耐心。

**无障碍**：
- `role="dialog" aria-modal="true"` + 焦点陷阱。
- 打开时焦点自动到关闭按钮（让孩子/键盘用户第一步就能退出）。
- 关闭后焦点回到触发元素（来源卡片）。
- 图片 alt 描述卡片主体（如"青铜鼎，夏朝礼器，三足两耳"）。

### 4.3 滚动与溢出

- 卡片详情长文本时允许内部滚动，不影响外层遮罩固定。
- 移动端开启 `overscroll-behavior: contain`（防止拉到底滚到 body）。

## 5. 朝代图谱的导航手势与键盘操作

### 5.1 移动端手势

- **纵向滑动**：滚动浏览 10 关卡片（默认原生滚动）。
- **点击卡片**：进入对应关卡或触发重玩确认。
- **长按**（可选增强，非必做）：展开朝代简介 tooltip；不做也不影响核心功能。
- **禁止**：pinch zoom、复杂 swipe（孩子不熟悉）。

### 5.2 桌面端键盘与鼠标

- **Tab 顺序**：Logo → 语言 → 主题 → 卡册 → 错题入口（如有）→ 关卡 1 → 2 → ... → 10。
- **方向键**：在关卡网格内移动（Grid 按 5×2 布局时，上下键跨行，左右键跨列）。
- **Enter/Space**：激活当前焦点项。
- **鼠标 hover**（桌面端增强）：
  - Unlocked 关卡：200ms 延迟后弹出 tooltip 显示朝代简介（40-60 字）+ "已通关 X/5"（如有）。
  - Locked 关卡：tooltip 显示"通关 XX 朝就解锁啦"。
  - tooltip 不替代正式交互，仅增强。

### 5.3 响应式切换动效

- 从移动端拖到桌面宽度（用户调窗口）时，布局从纵向流 → 网格的过渡用翻页感 300ms，卡片保持身份（不跳跃重排）。

## 6. 错题回看（`/wrong[/:dynastyId]`）

### 6.1 布局与入口

**触发条件**：首页错题入口、结算页"再来看一眼"。
**布局**：
- 顶栏：返回箭头 + 标题"再来看一眼"。
- 筛选器：
  - 移动端：下拉 "全部 / 夏 / 商 / ..."（仅显示有错题的朝代）。
  - 桌面端：水平 Tab。
- 列表：每条一张卡片，显示题干预览 + 朝代 tag。

### 6.2 展开交互（儿童友好核心决策）

**决策**：**inline expand**（点击卡片后在原位展开），不进新页面、不用手风琴（避免多题同时展开混乱）。
- 点击卡片 → 发现感 400ms 展开：题干完整 + 我的选项（标"我选的"，不标"错"）+ 正确选项（标"正确答案"，勾 icon）+ 解释文字。
- 展开时自动折叠其他已展开项（单次只看一题，降低认知负荷）。
- 再次点击已展开项 → 翻页感 200ms 折叠。

**儿童友好微交互**：
- 展开的卡片顶部有"这题的答案在这里"引导文本，不说"你答错了"。
- 选项区"我选的" vs "正确答案"并排展示，色彩为 semantic.neutral + semantic.success（**不用 attention 色强调"错"**，只强调"对"）。
- **禁止"重答"按钮**（page-map IA-6 明确：只看不做）。

**空态**：
- `wrongQuestions.length === 0`：显示插画 + "太棒了，暂时没有要再看一眼的题～"
- 深链访问 `/wrong/:dyn` 但该朝代无错题：显示"XX 朝没有要再看的题，换个朝代看看？"+ "看全部"按钮。

### 6.3 键盘与无障碍

- Tab 顺序：返回 → 筛选器 → 列表项 1 → 2 → ...
- Enter/Space 展开/折叠。
- 展开的内容 `aria-expanded="true"`，`aria-live="polite"` 播报题干和正确答案。
- 屏幕阅读器播报文本避免"错误"用词："这题的正确答案是 B，解释是..."

## 7. 语言切换与主题切换的补充规则

### 7.1 语言切换

- **切换时机约束**：答题页禁切换（顶栏入口已移除）。
- **切换范围**：所有静态文案 + 数据里的 `_zh`/`_en` 字段；URL 不变；滚动位置不变。
- **过渡动效**：100ms 淡出 + 200ms 淡入（翻页感），`aria-live` 播报"已切换到 English"。
- **英文膨胀预算**：所有按钮/标题容器宽度 +30-60% 预留（i18n 硬约束）。
- **错误容错**：某条 key 缺失翻译时显示中文原文 + 小 warning icon（开发环境）；正式环境静默回退到中文。

### 7.2 主题切换

- **切换范围**：`<html data-theme="light|dark">` 切换；所有 semantic token 重新插值。
- **过渡动效**：全站 300ms 颜色插值（CSS transition on root tokens）；单独保证图片/插画不闪（图片已用 `prefers-color-scheme` media 匹配双套资源时，用 opacity crossfade 代替硬切）。
- **防闪**：r-frontend 需在 `<head>` 同步脚本预读 localStorage theme，避免 FOUC。
- **跟随系统**：监听 `prefers-color-scheme` 变化，`theme === 'system'` 时实时跟随。
- **触发位置**：首页顶栏 + 卡册页顶栏 + 结算页顶栏；答题页允许切换（不影响数据）。
